## Key Architecture Decisions

- **No React Query / Axios** — Next.js 16 Server Actions + `useActionState` used throughout; data fetching in client pages via `fetch()` with `document.cookie` token
- **httpOnly cookie** (`auth_token`) — JWT stored server-side only; client components read it via `document.cookie` for API calls, never JS-accessible for security
- **Browser-safe JWT decode** — `atob()` in `shared/lib/jwt.ts`; `Buffer` only used in server-only `session.ts` and `proxy.ts`
- **`useActionState` initial state** — `null` (not `undefined`) for actions that return extended state types (`TenantFormState`, `CreateUserFormState`)
- **Modular monolith** — cross-module calls go through service interfaces only (never cross-module repository access)
- **Soft deletes** — `deleted_at` on `appointments`, `patients`, `subscriptions`
