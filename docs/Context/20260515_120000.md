## Key Technical Decisions
- JWT stored in httpOnly cookie named `auth_token` (set via Server Action)
- Next.js 16 patterns: Server Components for display, Client Components for interaction, Server Actions for mutations
- No React Query (server components handle data fetching natively in Next.js 16)
- No Axios (fetch with typed helpers in client.ts)
- Zod for validation in server actions
- Spring Security with jjwt 0.12.6 for JWT
- springdoc-openapi 2.8.8 for Swagger UI

## Important Notes
- Backend context path: /api (server.servlet.context-path=/api)
- So auth endpoint is POST /api/auth/login (frontend calls /auth/login via proxy)
- SUPER_ADMIN has no tenantId (nullable in users table)
- Walk-ins: appointment_time can be null, is_walkin=true
- Appointments cannot be modified after CANCELLED status
- All queries must filter by tenant_id except SUPER_ADMIN
