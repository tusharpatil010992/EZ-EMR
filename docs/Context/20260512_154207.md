### Architecture Rules Followed
- No cross-module repository access вЂ” modules communicate via service layer only
- Every entity has `tenant_id UUID NOT NULL` (multi-tenancy)
- Walk-in handled via `is_walkin` flag on both `Patient` and `Appointment` вЂ” no separate system
- Single PostgreSQL DB, Flyway migrations at `db/migration/`
- REST only, no GraphQL/RPC

### Database

**Start DB:** `docker-compose up -d`

Connection (default):
- Host: localhost:5432
- DB: emr_db
- User: emr_user
- Password: emr_password

Flyway migration: `V1__init_schema.sql` вЂ” creates `patients`, `appointments`, `subscriptions` tables with indexes.
